# アプリケーションのアーキテクチャ概要

このドキュメントは、「Kawaii Agent」アプリケーションの技術的な仕組みとアーキテクチャについて解説します。

## 1. アプリケーションの目的

このアプリケーションは、AIを搭載したインタラクティブなデスクトップマスコットです。ユーザーは3Dキャラクター（VRM形式）をデスクトップに表示し、音声やテキストを通じて対話を楽しむことができます。

## 2. 主要技術スタック

- **アプリケーションフレームワーク:** **Electron**
  - Web技術（HTML/CSS/JavaScript）を用いて、Windows/macOSで動作するクロスプラットフォームなデスクトップアプリを構築します。
- **UIライブラリ:** **React**
  - `App.jsx`をルートとするコンポーネントベースでUIを構築しています。
- **3Dレンダリング:** **Three.js** + **@react-three/fiber**
  - 3Dシーンの宣言的な構築を可能にし、Reactコンポーネントとして3Dオブジェクトを扱います。
- **VRM表示・アニメーション:** **@pixiv/three-vrm**, **@pixiv/three-vrm-animation**
  - VRMモデルの読み込み、操作、および `.vrma` 形式のアニメーション再生を担います。
- **AI/LLM連携:**
  - **@mlc-ai/web-llm:** ローカル環境でLLMを実行します。
  - **OpenAI, Replicate:** クラウドベースのAIモデル（GPT、Stable Diffusionなど）と連携します。
- **音声認識・合成:**
  - **Vosk, VAD:** マイクからの音声をリアルタイムで認識します。
  - **VOICEVOX, MoeTTSなど:** テキストから音声を生成します。
- **ビルドツール:** **Webpack** + **Babel**
  - JSXや最新のJavaScript構文を、Electronのレンダラープロセスで実行可能なコードに変換・バンドルします。

## 3. アーキテクチャの概要

本アプリケーションは、Electronの標準的な**メインプロセス**と**レンダラープロセス**の2層構造で構成されています。

- **メインプロセス (`main.js`)**
  - アプリケーションのライフサイクル（起動、ウィンドウ作成、終了）を管理します。
  - OSネイティブな機能（ファイルアクセス、メニュー作成など）を担当します。
  - `preload.js` を介して、レンダラープロセスに安全な形で特定のNode.js APIを公開します（IPC通信）。

- **レンダラープロセス (`renderer.js`, `App.jsx`, etc.)**
  - アプリケーションのUI全体を描画します。`index.html` をホストとし、ReactがUIを構築します。
  - 3Dキャラクターの表示 (`VRMViewer.jsx`) や、ユーザーとのインタラクション（ボタンクリック、テキスト入力）を処理します。
  - `services`を通じて、AI対話や音声合成などのバックグラウンド処理を呼び出します。

## 4. プロジェクト構造

主要なディレクトリとファイルの役割は以下の通りです。

```
Kawaii_Agent/
├── src/                # アプリケーションのソースコード
│   ├── components/     # Reactコンポーネント群 (VRMViewer, 各種モーダルなど)
│   ├── services/       # 外部APIやバックエンド機能との連携ロジック
│   ├── main.js         # Electron メインプロセス
│   ├── renderer.js     # Electron レンダラープロセス (Reactの起動)
│   └── App.jsx         # Reactのルートコンポーネント
├── bvh/                # Mixamoなどから取得した元のアニメーションデータ (BVH形式)
├── vrma/               # アプリケーション用に変換されたアニメーションデータ (JSON形式)
├── scripts/            # 補助的なスクリプト
├── convert_*.mjs/py    # アニメーション変換用スクリプト
├── package.json        # 依存関係と実行スクリプトの定義
└── webpack.config.js   # Webpackのビルド設定
```

## 5. 処理フロー

1.  **起動:** `npm start` コマンドでElectronが起動し、`main.js` が実行されます。
2.  **ウィンドウ生成:** `main.js` がブラウザウィンドウを生成し、`index.html` を読み込みます。
3.  **React起動:** `index.html` から `renderer.js` が実行され、Reactアプリケーションが `<App />` コンポーネントをDOMにマウントします。
4.  **VRM表示:**
    - `App.jsx` 内に配置された `<VRMViewer />` コンポーネントが、指定されたVRMモデルを読み込み、Three.jsのシーン上に表示します。
5.  **ユーザーインタラクション:**
    - ユーザーがUIを操作（例: テキスト入力、ボタンクリック）すると、イベントが発火します。
    - `aiService.js` などのサービスが呼び出され、LLMとの対話処理が実行されます。
    - LLMからの応答テキストは `voicevoxService.js` などに渡され、音声が生成・再生されます。
    - 同時に、応答内容に応じたアニメーションが `vrma/` ディレクトリから読み込まれ、VRMモデルに適用されます。

## 6. アニメーション変換パイプライン

本アプリケーションは、多様なモーションをVRMモデルで再生するために、独自の変換パイプラインを持っています。

1.  **元データ:** `fbx/` や `bvh/` に、Mixamoなどで入手したアニメーションファイルを配置します。
2.  **変換:**
    - `convert_fbx_to_bvh.py`: FBX形式をBVH形式に変換します（必要な場合）。
    - `convert_bvh_to_vrma.mjs`: BVHファイルを、`@pixiv/three-vrm-animation` が解釈できる `.vrma` 形式のJSONファイルに変換します。
3.  **利用:** アプリケーションは、この変換済みの `.vrma` ファイルを `vrma/` ディレクトリから読み込んで、キャラクターのアニメーションとして再生します。
