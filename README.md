# DesktopCharacter
### コンセプト
情報化社会になってパーソナライズされた情報を取得、利用するようになったが、その人本人が自分についての理解を深める機会・時間が不足・欠如してると感じる。
デスクトップ上に常駐してユーザ情報を監視、記録しながらパーソナルナレッジマネジメントの助けになるデータを記録していく（日記機能やObsidianとの連携）

### 昨日
デスクトップにキャラクターを設置、会話できるようにする。  
目標：ユーザの作業画面に対して反応する。  
（仕事の助けじゃなくてあくまでにぎやかし。力にはなれないけど身を案じてくれる感じ。  
作業時間が長かったら気分転換や休憩を提案、Youtubeだったらちょっと感想言ったり、調べ物のアシストしたり。
）


### プログラムの構成考え中
- main  
プログラムの初期化、設定の読み込み等
- textsystem  
geminiからのリマインドや感情のコマンドの読み分け、characterへの更新,UIのUpdate, Calenderからタスクや予定の取得・追加
- gemini  
geminiAPIをたたいてセリフを取得、また、コマンドを指示した形で返してもらってキャラの状態変更
- character  
キャラクターの立ち絵や声、そのほかステータスの管理、保持
- VoiceVox  
キャラクターの音声生成
- UI  
ユーザへの出力
- Calendar  
GoogleCalendar,Taskから予定を取得、場合によっては追加によって現状の記録を残す。
